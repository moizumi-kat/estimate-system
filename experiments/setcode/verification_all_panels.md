# 全盤種 検証サマリ（本物の見積書ベース＋実API）

2026-07 セッション。**人の見積書を正解**に、全盤種（配電盤・制御盤・分電盤・弱電端子盤）で選定を検証した記録。
対象案件：とくなが病院・船引小野・東部学校給食・八戸酒造（見積書あり）、木村病院・尼崎西向島（図面あり）。

## 詳細

### 1. 配電盤（受変電）＝セットコード方式（本セッションの主成果）
- 積算コード表から **11系(高圧)・16系(段積)・17系(低圧) 148件** をDB追加候補として作成。db.json欠落を補完。
- **選定は決定的**（盤種＋相＋容量＋計器＋VCB＋操作の完全一致、容量切上）。AI推測なし。
- **確認ゲート**：計器種別は単線図で確定しづらいため**常に人が確認**（コンボボックス・既定=普通角）。
- 検証：
  - 見積書4案件の低圧TR **20/20** がセット候補に命中（とくなが8・船引4・東部3・八戸5）。
  - 実API 図面→抽出→セレクタ：木村 低圧7/7◎＋高圧受電/饋電2/2◎、東部 相容量3/3、八戸 相容量5/5、尼崎 相容量+計器読取。
  - 相・容量の抽出は正確、計器は安全にゲート（推測させない）。

### 2. 分電盤（電灯分電盤）＝60系
- 見積書4案件の分岐MCB/ELB **537/547（98%）** を既存select_oneが正しく選定。配電盤と異なり元々強い。
- 未選定10件は表記ゆれ等の軽微。改善余地は小。

### 3. 弱電端子盤＝62系（改善実施）
- 問題：見積書は「端子盤」語なしで「10P(T付)」「10P(E1,E2,E3,トクE)」「MDF 10P」「TB 15A 10P」等と表記→既存select_oneが **0%**。
- 改善 `terminal_select.py`：極数P＋端子指標で 端子盤(端子付/無)・接地端子盤(629)・MDF(622)・保安器函(628) を最近傍上位選定。遮断器の3P・コンセント2Pは除外（誤認防止）。
- 結果：全案件 **0→27/27（100%）**。

### 4. 制御盤（動力制御盤）＝22-29系 分岐回路
- 見積書の負荷（「2.2KW (L-S)」「INV n KW」等）は分岐回路コードに選定可。
- 未選定は AC/DCリアクトル等＝**INV分岐回路コードにセット内包**される部品（配電盤セット内包品と同様、個別選定不要）＝実質問題なし。
- ⚠️ ただし「回路パターン集＋適用表→分岐回路コード」の**パターン方式（poc_control_v2）はapp.py未統合**（PoC止まり）。制御盤全体の精度確定には統合が必要（引き継ぎ書 第8章の未完了タスク）。

### 重要な結論：選定ロジックは強い、ボトルネックは抽出
- **見積書ベース（＝抽出が正しい前提の選定ロジック）**：分電盤98%・制御盤(負荷)OK・端子盤100%。
- **実パイプライン（図面→抽出→選定）**：分電盤75%・制御盤51%・端子盤12%。
- 差＝**抽出の難しさ**。特に：
  - **制御盤**：回路パターン方式（poc_control_v2）未統合が主因。統合すれば大幅改善見込み。
  - **端子盤**：図面抽出で**極数(NP)や種別が読めないと△**（例「弱電端子盤 T-1」だけで極数なし）。選定ロジックは改善済（見積書100%）だが、**極数を確実に読む/読めなければ確認ゲート**が必要（配電盤の計器と同じ構造）。terminal_selectは「電話20P/放送30P」等の図面表記にも対応済。弱電機器（アンテナ/HUB/分配器/カメラ）は見積対象外として除外。
