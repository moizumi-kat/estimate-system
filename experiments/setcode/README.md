# setcode/ — 配電盤セットコード（設計・候補・セレクタ）

積算コード表（御社正式マスタ）の**セットコード方式**を、抽出→選定システムに載せるための設計・DB追加候補・選定ロジック一式。
2026-07 セッションで作成。人（茂泉様）による照合・承認を前提とした**候補**であり、承認後にDB(db.json)へ反映する。

## 背景：なぜセットコードか
人の見積作業は「部品を1点ずつ拾う」のではなく、**盤種＋数個の仕様でセットコードを1つ選ぶ→積算ソフトが部品一式へ展開・積算**する。
DB(db.json)には制御盤の分岐回路セット(22-29系)と非常用(30系)は入っていたが、**配電盤のセットコード(11/16/17/19系)が丸ごと欠落**していた（木村病院で配電盤が0〜19%だった根本原因）。本ディレクトリはこの欠落を埋める。

## システムの役割と出力（確定）
```
図面 → 【抽出】盤属性 → 【確認ゲート】読切れない仕様を人に問う → 【セレクタ】セットコード確定
                                                              → 積算ソフトへ入力(展開・積算はソフト側)
```
- 出力＝**セットコード**（＋制御盤は分岐回路22-29、非セット個別品は個別コード）。
- **展開(expand)・単価はシステムの責任外**（積算ソフトが持つ）。expandは参考・検証用にのみ保持。
- 選定は**決定的**（属性の完全一致＋容量は仕様以上で最近傍上位＝切上）。AIはコードを生成しない。

## 確認ゲート（抽出と選定の間）
抽出で**確信を持って読めた仕様のみ**採用し、**読み切れない仕様は選定前に一括で人に確認**する（推測しない＝◎誤答ゼロ）。
- 計器種別の**標準＝普通角**（既定値として提示）。
- 対象は計器種別に限らず、容量が不鮮明・VCB操作方式が不明瞭 等あらゆる読取不能仕様。
- **UIはコンボボックス（プルダウン）で選択肢を提示**し、自由入力させない（既定値を初期選択）。選択肢は `setcode_selector.py` の `OPTIONS`。
- `needs_confirm()` が不足仕様を、`run()` が各仕様の `options`＋`default` を返す（UIがコンボボックス描画）。`apply_defaults()` は確認を省く場合の既定当て。

### 選定属性（新設計フィールド）
- 共通: `settype`（低圧/高圧/高圧LBS/段積/段積VCS）
- 低圧(17): `meter`(広角/普通角/マルチ)・`phase`(1φ3W/3φ3W/スコット)・`cap`(容量KVA)
- 高圧(11): `role`(受電盤/饋電盤)・`meter`・`vcb`(8KA/12.5KA)・`op`(手動/電動)
- 段積(16): `role`(受電盤/一段積/二段積/三段積/母線連絡/母線連絡+一段積)・`meter`・`vcb`・`op`(電動引出)
- 段積VCS: `role`・`op`(電磁/電磁引出PF)

### コード符号の要点
- 11系: `11 [計器0/3/6] [盤種+VCB 3受電8K/4受電12.5K/7饋電8K/8饋電12.5K] [操作1手動/2電動]`
- 16系: 広角基準、`普通角=下3桁+100`・`マルチ=+200`。VCB=電動引出。段積=一段積等の組合せ。
- 17系: `17 [計器0広角/1普通角/2マルチ] [相・容量連番]`。全て**TR支給**前提。

### 展開(expand)の要点（参考・検証用）
- 17系: FL・LBS・TR支給・LG-RY＋計器(広角/普通角=VM/AM/VS/AS、マルチ=マルチ指示計)＋CT・LUG(容量連動)。スコットは計器/CT/LUG×2。
- 11系: 手動=計器＋VCB(手動)＋OCR等。**電動=手動＋CS×1＋AUX-RY×4**(p8脚注)。受電盤=全計器/饋電盤=AM・AS。
- 16系: p8の受電(col3)/一段積(col4)/母連(col5)/VCS(col6)を組合せ。二段積=一段積×2、三段積=一段積×3、母連+一段=母連+一段積。1ヶSP=1段スペース(機器なし)。
- CT変流比・LUG容量・VT VA は容量/図面依存 → 図面読取＋二次電流ルール、迷えば△（`44104?`等の仮置き）。

## 検証（木村病院）
`setcode_selector.py` を実行。単線結線図(E-004)からTRの相・容量を読取り→セレクタで：
- 低圧系7盤（電灯3・動力2・スコット2）で **図面→セットコード 7/7 ◎**。
- 計器種別は当案件では全てマルチ（標準は普通角、案件標準/器具表で決定）。

## DB照合の副産物（`db_reconcile_findings.md`）
積算コード表 全52ページ を db.json と全件照合。個別部品(40-98系)は名称・単価ほぼ完全一致。
欠落＝配電盤セット122＋制御盤2P用22＋個別31（90系板金15・97系認定4・40系4P5・52456・31905/6 等）。
既存見直し＝50系欠相保護無の単価空欄10・43630単価・30050系「1種/2種」ラベル（**単価は積算ソフト管轄のため優先度低**）。
